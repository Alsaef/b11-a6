Frequently Asked Questions

the difference between var, let, and const?
var is function-scoped, allows re-declaration and re-assignment, and is hoisted with undefined.
let is block-scoped, does not allow re-declaration but allows re-assignment.
const is block-scoped, does not allow re-declaration or re-assignment, and its value remains constant.
the difference between map(), forEach(), and filter()?
map() returns a new array with modified elements without changing the original array.
forEach() executes a function for each element but does not return a new array.
filter() returns a new array with elements that meet a specific condition.
explain arrow functions and how they are different from regular functions?
Arrow functions use => for a shorter syntax, inherit this from the surrounding scope, support implicit
return, cannot be used as constructors, and lack the arguments object, unlike regular functions.
how JavaScript Promises work?
JavaScript Promises handle asynchronous operations. A Promise has three states: pending, resolved
(fulfilled), or rejected. It executes asynchronously and uses .then() for success and .catch() for errors.
how closures work in JavaScript?
Closures in JavaScript occur when a function retains access to its lexical scope, even after the outer
function has finished executing. This allows the inner function to access variables from the outer
function, even after it's returned.
